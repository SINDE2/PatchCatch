[오프닝: 프로젝트 소개] (1페이지)
안녕하십니까. 'ZeroDay Patch Monitor(제로데이 패치 모니터)' 프로젝트 발표를 맡은 [발표자 성함]입니다. 

저희 프로젝트는 **'실시간 게임 패치노트 자동화 시스템'**으로, 단 48시간이라는 제한된 시간 동안 챌린지 형태로 구현한 Python Flask 기반의 웹 서비스입니다. 



게이머들에게 가장 중요한 정보인 패치노트를 어떻게 하면 가장 빠르고 편리하게 전달할 수 있을까라는 고민에서 시작된 프로젝트입니다.

[배경: 문제점 인식] (2페이지)
프로젝트를 시작하게 된 배경에는 크게 세 가지 문제점이 있었습니다.

첫째, 정보의 파편화입니다. 전 세계적으로 방대한 게임 정보가 여러 곳에 분산되어 있어, 게이머와 개발자 모두 중요한 업데이트를 놓치기 쉽습니다.  둘째, 언어 장벽입니다. 많은 주요 패치노트가 영어로 먼저 공개되는데, 이는 국내 사용자들의 정보 접근성을 떨어뜨립니다.  셋째, 서버 간 업데이트 시차입니다. 국내외 서버 간의 시차로 인해 발생하는 정보 불균형은 사용자 경험을 저하시키는 주된 원인이 됩니다. 



[솔루션: PatchCatch 개요] (3페이지)
저희는 이러한 문제를 해결하기 위해 'PatchCatch(패치캐치)' 솔루션을 개발했습니다. 

이 시스템은 크게 4가지 핵심 기능으로 구성됩니다.


웹 스크래핑: Requests와 BeautifulSoup을 이용해 실시간으로 정보를 수집하고, 


자동 번역: Googletrans 라이브러리로 영문 패치노트를 즉시 한글화합니다. 


데이터 저장 및 비교: JSON DB를 활용해 버전을 비교하여 새로운 패치를 감지하며, 


이메일 알림: smtplib를 통해 감지된 새 소식을 사용자에게 즉시 메일로 알립니다. 

이 모든 과정을 48시간 안에 구현하여 게임 정보의 수집부터 전달까지 완벽하게 자동화했습니다. 


[아키텍처: 데이터 파이프라인] (4페이지)
시스템 아키텍처를 보시겠습니다. 저희는 효율적인 개발을 위해 MVC 패턴 기반의 Flask 프레임워크를 채택했습니다. 

데이터 파이프라인은 명확합니다. 먼저 '수집' 단계에서 게임사 웹사이트의 데이터를 긁어오고, '가공' 단계에서 정규표현식과 번역 API를 통해 데이터를 정제합니다. 이후 '저장' 단계에서 파일 기반의 JSON DB에 데이터를 적재한 뒤, 마지막 '알림' 단계에서 사용자에게 이메일을 발송하는 구조입니다. 




[핵심 기술: 필터링 및 비교 로직] (5페이지)
이 시스템의 기술적 차별점은 **'정확도'**에 있습니다.

게임사 공지사항에는 단순 이벤트나 잡다한 정보가 혼재되어 있습니다. 저희는 이를 걸러내기 위해 **'이중 검증 필터링 로직'**을 구현했습니다. 코드에서 보시는 것처럼 'Patch'와 'Notes' 키워드는 포함하되, 'TFT'와 같은 불필요한 키워드는 제외하는 방식입니다. 


또한, 정규표현식을 활용한 버전 비교 알고리즘을 개발했습니다. 단순 텍스트 비교가 아니라, 버전 숫자(float)를 추출하여 북미 서버와 한국 서버의 수치를 수학적으로 비교함으로써 동기화 상태를 정확히 판별합니다. 



[킬러 피처: LoL 서버 비교] (6페이지)
저희 프로젝트의 **킬러 피처(Killer Feature)**인 'LoL 서버 비교' 기능입니다. 

화면을 보시면 북미(NA) 서버는 14.23 버전인 반면, 한국(KR) 서버는 14.21 버전임을 한눈에 알 수 있습니다. 시스템은 이를 분석하여 '북미 선행 공개' 상태임을 알려주고, 한국 서버는 '동기화 대기' 중임을 시각적으로 보여줍니다. 




이를 통해 게이머들은 정보의 시차를 극복하고, 남들보다 한발 앞서 정확한 게임 전략을 수립할 수 있습니다. 



[최종 결과물: 대시보드 UI] (7페이지)
최종 결과물인 대시보드 화면입니다. 


Bootstrap 프레임워크를 사용하여 모바일과 PC 어디서든 볼 수 있는 반응형 디자인을 구현했습니다. 새로운 패치가 감지되면 해당 게임 카드에 자동으로 "[NEW]" 배지가 활성화되어 사용자가 즉시 인지할 수 있으며, 앞서 설명한 서버 버전 비교 정보가 직관적인 테이블 형태로 제공됩니다. 



[결론 및 향후 계획] (8페이지)
발표를 마치며 요약하겠습니다.

저희 팀은 48시간이라는 짧은 시간 동안 **'수집-가공-저장-알림'**에 이르는 풀스택 자동화 파이프라인을 성공적으로 구축했습니다. 이를 통해 정보 파편화와 언어 장벽, 그리고 시차로 인한 불균형 문제를 해결했습니다. 

향후에는 LoL과 발로란트, 이터널 리턴 외에 더 많은 게임으로 서비스를 확장하고, 현재의 이메일 알림을 넘어 모바일 푸시 알림까지 연동하여 사용자 경험을 더욱 강화할 계획입니다. 


이상으로 'ZeroDay Patch Monitor' 발표를 마치겠습니다. 감사합니다.
